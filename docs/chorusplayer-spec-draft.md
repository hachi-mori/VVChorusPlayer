# ChorusPlayer 機能仕様ドラフト（実装前）

最終更新: 2026-03-10
対象: VVChorusPlayer の次期「合唱特化」機能

## 1. このドキュメントの目的

- アイデアを実装可能な仕様に整理する
- UI と内部処理の責務を分けて定義する
- 実装優先度を明確にして段階導入しやすくする

## 2. 用語と命名（提案）

操作モード名は以下を提案します。

- `選択モード`（Selection Mode）
- `手動選択`（Manual）
  - ユーザーがキャラ一覧から直接選ぶ
- `自動編成`（Auto Arrange）
  - ルールに従ってキャラを自動で選ぶ

補足:
- 「オート」より「自動編成」の方が、何を自動化するかが明確
- タブ名は `手動選択` / `自動編成` が分かりやすい

## 3. 全体要件（今回の仕様範囲）

### 3.1 必須

- 手動選択と自動編成を切り替え可能
- 自動編成時は「先頭スタイルのみ」を使用
- 合唱人数（1〜最大）をノブ（回転スライダー）で指定
- ステレオ幅（Pan Width）をノブで指定
- 選択方式を切り替え可能
  - ランダム
  - 声質
  - 音域
- 音域調整量（keyshift）の重複を可能な範囲で回避

### 3.2 優先度低

- ヒューマナイズ（ノート長やタイミングの微小揺らぎ）
- リバーブ
- EQ
- 生成高速化の追加工夫

## 4. UI 仕様案

## 4.1 モード切替

- キャラ選択エリア上部にタブ配置
  - `手動選択`
  - `自動編成`
- タブ切替で表示コントロールを切り替える

## 4.2 手動選択タブ

- 現在のキャラ一覧（複数選択）を利用
- 全選択 / 選択解除ボタンは継続
- 「全スタイル表示」トグルは利用可能

## 4.3 自動編成タブ

表示コントロール:

- `合唱人数` ノブ
  - 範囲: 1〜N（N = 利用可能キャラ数）
- `広がり` ノブ（Pan Width）
  - 0〜100%（内部は pan 値へ変換）
- `選択方式` セレクタ
  - ランダム / 声質 / 音域
- `声質タイプ` セレクタ（声質方式選択時のみ有効）
  - 女声 / 男声 / ソプラノ / アルト / テノール / バス

補足:
- 自動編成時は `全スタイル表示` を無効化し、先頭スタイル固定
- 音域ずらし（keyshift重複回避）は常時 ON（UIトグルなし）

## 5. Pan Width 仕様

## 5.1 配置ルール

- 人数 N に対し、中央対称で pan を割り当てる
- 例:
  - 3人: -10, 0, +10
  - 4人: -15, -5, +5, +15

## 5.2 一般化

- 幅パラメータを `W`（最大絶対 pan）とする
- N 人のインデックス i=0..N-1 に対し、等間隔配置
- 中央を 0 にして左右対称に配分

実装イメージ:
- N=1: [0]
- N>=2: `linspace(-W, +W, N)` に近い配置

## 5.3 幅ノブとの対応

- ノブ値 0〜100% を `W=0〜Wmax` へマップ
- 初期値案: 40%（軽く広がる）
- `Wmax` はまず ±30 程度から検証

## 6. 自動編成ロジック仕様

## 6.1 共通前提

- 候補は「各キャラの先頭スタイルのみ」
- 人数 K をノブで指定
- 候補数 < K のときは候補数を上限にする

## 6.2 方式A: ランダム

- 候補集合から重複なしランダム抽出
- seed 固定オプションは将来拡張

## 6.3 方式B: 声質

仕様:

- キャラごとに `女声` / `男声` / `ソプラノ` / `アルト` / `テノール` / `バス` の判定を持つ
- 判定しきい値:
  - バス: `keyshift <= -20`（超低音の特別枠）
  - テノール: `-19 <= keyshift <= -12`（低め男声帯）
  - アルト: `-11 <= keyshift <= -5`（低め女声〜中音域）
  - ソプラノ: `keyshift >= -4`（明るい高め女声）
  - 男声: `keyshift <= -12`
  - 女声: `keyshift >= -11`
- 選択タイプに一致する候補から優先抽出
- 足りない場合は反対グループから補完

実装メモ:
- 初期は `Source/keyshift_table.json` の keyshift 値から判定
- 将来、VOICEVOXメタ情報等から自動推定を検討

## 6.4 方式C: 音域

### 6.4.1 楽譜側の代表音算出

- 入力 vvproj の notes から代表音 `P_ref` を求める
- 採用方式: 音高の平均値（mean）
- 備考: 最高音と最低音の中央値（mid-range）は採用しない

### 6.4.2 キャラ適性スコア

- keyshift 0 が得意音域 G4 付近という前提
- keyshift +1 は G#4 寄り、-1 は F#4 寄り
- `P_ref` に合わせて、必要な補正量との距離が小さいキャラを優先

スコア例:
- `score = abs(requiredShift - characterKeyshift)`
- 小さい順に候補採用

## 7. 音域ずらし（keyshift重複回避）仕様

目的:
- 同じ keyshift のキャラが重なると同質化しやすい問題を軽減

ルール案:

- 選択後、keyshift 値の重複を確認
- 重複キャラには `±1` を優先して割当調整
- まだ重複する場合は `±2` まで許容
- 上限は `|delta|<=2` を基本（過補正回避）

注意:
- 完全重複回避は必須にしない（破綻回避を優先）
- 音質劣化が目立つ場合は調整を打ち切る
- この機能は常時有効（OFFにはしない）

## 8. ヒューマナイズ（優先度低）

目的:
- 機械的な同時発声を軽減して人間らしい厚みを作る

案:

- ノブで強さ調整（0〜100%）
- ノート開始/終了を微小に揺らす
- ただし以下を維持
  - 曲全体長は保持
  - 極端な発音崩れを避ける

導入順:
- まずは OFF 前提で設計のみ

## 9. エフェクト（優先度低）

## 9.1 リバーブ

- 合唱感の付与に有効
- ただし先にパンと人数制御を完成させる

## 9.2 EQ

- 帯域の衝突や高域ノイズ軽減に有効
- 自動EQは難しいため、最初はプリセット式が現実的

## 10. 生成高速化（調査メモ）

候補:

- キャラ別合成の並列化（同時HTTP）
- 中間データ再利用（同一条件時）
- セグメント分割戦略の最適化

注意:
- 並列化は Engine 側負荷と失敗率を要検証

## 11. 実装フェーズ提案

### Phase 1（先行実装）

- モード切替タブ
- 自動編成の基本（先頭スタイル固定）
- 人数ノブ
- Pan Width ノブ
- 選択方式: ランダム

### Phase 2

- 選択方式: 声質 / 音域
- 音域ずらし（重複回避）

### Phase 3

- ヒューマナイズ
- リバーブ / EQ
- 速度最適化
