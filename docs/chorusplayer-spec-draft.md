## 1. このドキュメントの目的

- 実装済み仕様と、これから実装する仕様を分けて管理する
- UI仕様と内部ロジック仕様を簡潔に共有する

## 2. 実装ステータス

### 2.1 実装済み

- 選択モード切替
  - `手動選択` / `自動編成`
- 自動編成の編成方式
  - `オート` / `ランダム` / `パート別`
- パート別の分類
  - `女声` / `男声` / `ソプラノ` / `アルト` / `テノール` / `バス`
- 歌唱人数ノブ
  - 1〜利用可能人数（パート別時は該当グループ人数を上限）
- keyshift 重複回避（常時ON）
  - `±1` 優先、次に `±2` を試す
- パン自動配置（常時ON）
  - 手動選択 / 自動編成の両方に適用
  - 2人: `±0.10`、3人: `±0.20`、4人以上: 最大 `±0.30`
- 音域オート選択
  - vvproj の平均音高（mean）から適合キャラを優先
- VOICEVOXClient の動的オクターブ補正
  - 入力楽譜の平均音高とキャラ keyshift から動的に補正量を決定

### 2.2 未実装（これから）

- ヒューマナイズ（ノート長/タイミング微揺らぎ）
- リバーブ
- EQ
- 生成高速化（並列化・キャッシュなど）
- パン幅の再設定UI（将来復活予定）

## 4. 現在の内部仕様

### 4.1 候補キャラの扱い

- 手動選択時: UI選択キャラを使用
- 自動編成時: 先頭スタイルのみを候補に使用

### 4.2 自動編成ロジック

- ランダム
  - 候補から重複なし抽出
- パート別
  - しきい値でグループ判定し、一致候補を優先
  - 不足時は他グループから補完
- オート
  - vvproj の代表音 `P_ref` を平均音高で算出
  - `requiredShift = round(P_ref - 67)`
  - `score = abs(requiredShift - singerKeyShift)`
  - score が小さい順に採用

### 4.3 パート判定しきい値（実装値）

- バス: `<= -20`
- テノール: `-19 .. -12`
- アルト: `-11 .. -5`
- ソプラノ: `>= -4`
- 女声: `>= -11`
- 男声: `<= -12`

### 4.4 keyshift 重複回避（常時ON）

- 選定後に重複確認
- 調整候補順: `+1, -1, +2, -2`
- 使用済み keyshift と衝突しない値を優先採用

### 4.5 パン配置（常時ON）

- 人数 N に対して左右対称の等間隔配置
- `N=1`: `[0]`
- `N>=2`: `[-W, +W]` を等間隔
- `W = clamp(0.1 * (N - 1), 0.0, 0.3)`

### 4.6 動的オクターブ補正（VOICEVOXClient）

- 基準音: `G4 = MIDI 67`
- `P_singer = 67 + keyShift`
- `diff = P_score - P_singer`
- `oct = clamp(round(diff / 12), -2, +2)`
- `octSemitone = -oct * 12`
- `keyShift` 補正と合成して score/query に適用
